# What I Learned

When I first started learning Linux, I honestly found it quite intimidating.

Coming from a Windows background, almost everything felt unfamiliar. Even navigating the terminal was something I had to get used to. At the beginning, I found myself copying commands without always understanding what they actually did.

As I spent more time practising, things slowly started to click.

I began to understand that Linux isn't about memorising hundreds of commands. It's about understanding the concepts behind them. Once I understood topics like the Linux file system, users and groups, permissions, processes and package management, the commands themselves became much easier to remember.

One of the biggest turning points for me was learning Bash scripting.

Writing my own scripts taught me how Linux commands work together and gave me much more confidence using the terminal. Things like loops, functions, variables, command substitution and user input stopped feeling complicated once I started building my own scripts instead of just reading about them.

Another important lesson was learning how Linux is configured.

Creating my own `.vimrc`, understanding `.zshrc`, switching from Bash to Zsh, installing Oh My Zsh and rebuilding my terminal environment from scratch helped me understand how configuration files work and why they're so important.

Rather than simply following tutorials, I started asking myself *why* something worked.

That simple change in mindset has probably helped me learn more than anything else.

Along the way I also learned that troubleshooting is a skill in itself.

Instead of assuming something was broken, I learned to read error messages carefully, check configuration files, verify my environment and solve problems one step at a time.

Some of the problems I solved while learning included:

- Configuring Oh My Zsh
- Troubleshooting shell aliases
- Understanding shell configuration files
- Setting up SSH authentication with GitHub
- Rebuilding my Linux environment using Ubuntu on WSL
- Learning the differences between Bash and Zsh
- Working with Ubuntu and comparing it with Fedora

One thing I'm particularly proud of is that I no longer feel afraid of breaking things.

If something goes wrong, I'm much more comfortable investigating the issue, understanding what happened and fixing it. In many cases I've learned more from troubleshooting than from simply following a course.

I've also developed the habit of documenting what I learn.

Writing notes in my own words has helped reinforce my understanding and has given me a personal knowledge base that I can return to whenever I need a refresher.

Looking back, Linux no longer feels like a completely different operating system.

It feels like a tool that I'm becoming increasingly comfortable with every day.

Although I know there's still a lot to learn, I now have a solid foundation that I can build on as I continue studying Git, Networking, Docker, AWS, Terraform, GitHub Actions and Kubernetes.

This is only the beginning of my DevOps journey, but it's been incredibly rewarding to see how much progress can be made through consistent practice, curiosity and a willingness to keep learning.
