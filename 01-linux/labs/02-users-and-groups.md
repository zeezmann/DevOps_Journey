# Lab 02 - Users and Groups

## Objective

Understand how Linux manages users and groups, and learn how permissions are applied through user and group membership.

---

## Environment

- Ubuntu (VMware Fusion)
- Ubuntu 24.04 (WSL)

---

## Commands Used

```bash
whoami
id
groups
sudo
adduser
useradd
passwd
groupadd
usermod -aG
userdel
```

---

## Tasks Completed

- Identified the current logged-in user.
- Displayed user and group information.
- Explored how Linux stores user accounts.
- Learned how to create and remove users.
- Created groups and added users to them.
- Used `sudo` to perform administrative tasks.

---

## Challenges

At first, it wasn't obvious why Linux separates users into groups.

After practising user and group management, I understood that groups provide a much easier way to manage permissions for multiple users instead of configuring each user individually.

---

## Result

Successfully explored Linux user and group management and gained a better understanding of how access is controlled across the operating system.

---

## What I Learned

Linux is designed as a multi-user operating system. Understanding users, groups and `sudo` gave me a much better appreciation of how Linux handles security and access control.
