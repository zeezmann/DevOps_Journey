# Lab 05 - Package Management

## Objective

Learn how Linux package managers are used to install, update, remove and manage software packages.

---

## Environment

- Ubuntu (VMware Fusion)
- Ubuntu 24.04 (WSL)

---

## Commands Used

```bash
sudo apt update
sudo apt upgrade
sudo apt install
sudo apt remove
sudo apt search
apt show
```

---

## Tasks Completed

- Updated the local package list.
- Upgraded installed packages.
- Installed software using APT.
- Removed installed software.
- Searched for available packages.
- Viewed package information.
- Compared Ubuntu's APT package manager with Fedora's DNF package manager.

---

## Challenges

One of the biggest things to remember was that different Linux distributions use different package managers.

Rather than trying to memorise every command, I focused on understanding that they all perform the same core tasks: installing, updating, removing and searching for software packages.

---

## Result

Successfully managed software packages on Ubuntu using APT and gained a better understanding of package management across different Linux distributions.

---

## What I Learned

Package managers simplify software installation by handling dependencies automatically.

Although the commands differ between distributions, the overall workflow remains very similar once the concepts are understood.

---

## Real-World Relevance

Keeping Linux systems updated is an important part of system administration and security.

As a future DevOps Engineer, I'll regularly install software, troubleshoot package issues and ensure systems remain up to date using the appropriate package manager for the distribution.
