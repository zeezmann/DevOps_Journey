# git add

## What Is It?

`git add` stages changes, preparing them to be included in the next commit.

---

## Syntax

```bash
git add .
```

---

## Why Is It Useful?

Only staged files become part of the next commit.

---

## My Experience

Throughout the Linux and Bash modules, I used `git add .` from the root of my `DevOps_Journey` repository after completing a logical section of work. This helped me understand the difference between modified files in the working tree and staged files waiting to be committed.

---

## Best Practices

- Stage related changes together.
- Review files before staging.
- Avoid staging unfinished work.
