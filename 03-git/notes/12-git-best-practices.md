# Git Best Practices

## Best Practices

- Commit small, logical changes.
- Write meaningful commit messages.
- Pull before starting work.
- Push regularly.
- Keep repositories organised.
- Use feature branches where appropriate.
- Review changes before committing.
- Never commit sensitive information.

---

## Why Best Practices Matter

Following consistent Git practices makes collaboration easier and keeps project history clean and understandable.

---

## My Experience

As this repository grew, I realised that Git is much more than a backup tool.

Using meaningful commits, pushing regularly and keeping my repository organised made it much easier to switch between my MacBook and work laptop without losing track of my progress.

These habits have become part of my normal workflow whenever I work on a project.
