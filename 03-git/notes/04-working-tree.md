# Working Tree

## What Is It?

The working tree is the current state of the files in your local repository. It includes any files that have been modified, added or deleted since the last commit.

---

## Why Is It Useful?

The working tree allows you to make changes safely before deciding which ones should become part of your project's history.

---

## My Experience

While documenting the Linux and Bash modules, I regularly edited multiple files before committing them. My working tree often contained updated notes, labs and projects, giving me the opportunity to review everything before staging the changes.

---

## Best Practices

- Review your changes before staging.
- Avoid leaving unnecessary modified files.
- Commit related changes together.
