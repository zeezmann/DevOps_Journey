# What I Learned

Before this module, I thought Git was simply a tool for uploading code to GitHub.

Working on this DevOps Journey completely changed that perspective.

Using Git every day taught me how version control fits into real software development. I learned how the working tree, staging area and commits work together, and how GitHub acts as the central repository while each computer maintains its own local copy.

One of the biggest lessons came from working across both my work laptop and my MacBook. Learning when to use `git push` and `git pull` helped me understand how Git keeps multiple environments synchronised without recreating files manually.

More importantly, Git changed the way I approach projects. Rather than thinking about individual files, I now think in terms of meaningful commits, clean history and incremental progress.

Git has become one of the most valuable tools in my workflow, and I now use it naturally whenever I start a new project.
