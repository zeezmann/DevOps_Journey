# git push

## What Is It?

`git push` uploads local commits from your repository to a remote repository such as GitHub.

---

## Syntax

```bash
git push origin main
```

---

## Why Is It Useful?

Pushing changes keeps the remote repository up to date, provides an off-site backup and allows work to be shared across multiple devices or with other developers.

---

## My Experience

After completing each major section of my Linux and Bash modules, I pushed my commits to GitHub immediately.

This became part of my workflow:

```text
Build

↓

Test

↓

git status

↓

git add .

↓

git commit

↓

git push
```

Knowing that my work was safely stored on GitHub gave me confidence to continue developing on either my MacBook (at home) or work laptop.

---

## Best Practices

- Always commit before pushing.
- Push regularly.
- Check your push completed successfully.
