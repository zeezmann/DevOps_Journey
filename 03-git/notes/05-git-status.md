# git status

## What Is It?

`git status` displays the current state of the working tree and staging area.

---

## Syntax

```bash
git status
```

---

## Why Is It Useful?

It shows which files have been modified, staged or are ready to commit.

---

## My Experience

`git status` became the command I used most throughout this DevOps Journey repository. Before every commit, I checked which files had changed to make sure I wasn't accidentally committing unfinished work. It also confirmed when my working tree was clean after a successful commit.

---

## Best Practices

- Run `git status` before committing.
- Check your working tree is clean after pushing.
- Use it frequently while working.
