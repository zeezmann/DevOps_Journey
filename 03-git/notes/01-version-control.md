# Version Control

## What Is It?

Version control is a system that records changes made to files over time. It allows developers to track history, restore previous versions and collaborate without overwriting each other's work.

---

## Why Is It Useful?

Without version control, managing changes across multiple files or computers becomes difficult. Git solves this by keeping a complete history of every commit.

---

## My Experience

Before learning Git, I often relied on manual copies of files as backups. While building this DevOps Journey repository, I experienced how Git tracks every meaningful change through commits, making it much easier to organise and revisit my work.

---

## Best Practices

- Commit regularly.
- Write meaningful commit messages.
- Keep commits focused on a single change.
