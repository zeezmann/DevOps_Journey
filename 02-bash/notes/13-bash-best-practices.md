# Bash Best Practices

Writing clean and consistent Bash scripts makes them easier to understand, maintain and troubleshoot.

Following good practices also reduces errors and improves reliability.

---

## Best Practices

- Use meaningful variable names.
- Keep functions small and focused.
- Add comments where necessary.
- Quote variables to prevent unexpected behaviour.
- Validate user input.
- Check exit codes.
- Avoid duplicating code.
- Use consistent indentation.
- Test scripts before deploying them.

---

## Why Best Practices Matter

Good coding habits make scripts easier to read, debug and extend as they grow in complexity.

---

## What I Learned

Following best practices made my scripts feel much more professional. Small improvements such as using descriptive variable names and organising code into functions greatly improved readability and maintainability.
