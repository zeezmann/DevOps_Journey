# Lab 02 - User Input

## Objective

Accept user input using the `read` command.

---

## Environment

- Ubuntu 24.04 (WSL)
- macOS Terminal

---

## Commands Used

```bash
read
read -p
```

---

## Tasks Completed

- Prompted users for input.
- Stored responses in variables.
- Displayed entered values.

---

## Challenges

Understanding the difference between prompting users and using hardcoded values.

---

## Result

Created interactive scripts that respond to user input.

---

## What I Learned

Interactive scripts are far more flexible than scripts containing fixed values.

---

## Real-World Relevance

User input is commonly used when requesting usernames, directories, filenames and menu selections.
